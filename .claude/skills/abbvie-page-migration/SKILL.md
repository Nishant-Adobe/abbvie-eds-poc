## AEM Platform C Page Structure

Every AbbVie brand page follows this DOM structure:
```
<body>
  [Brand Explorer bar]          ← HCP sites only
  [Header (abbv-header-v2)]     ← Sticky, with safety line
  <main>
    [background-container.parbase]*  ← Each = one EDS Section
      [abbv-background-container]
        [abbv-background-container-display]  ← Background image/video
        [abbv-background-container-content]  ← Content overlay
  </main>
  [Inline ISI (abbv-inline-use-isi)]  ← Full safety info
  [References]                  ← Ordered list
  [Footer (abbv-footer)]
  [Floating ISI (abbv-safety-bar)]    ← Sticky bottom bar
  [Modals]                      ← Exit, study design, contact
  [Cookie banner (OneTrust)]
</body>
```

**Key rule:** Each `background-container.parbase` in `<main>` maps to **one EDS Section**.

## Migration Steps

### Step 1: Analyze Page Structure
Use the page analysis report at `/workspace/Analysis/page-analysis-report.html` to identify which blocks appear on the target page.

### Step 2: Map Sections
Each `background-container.parbase` becomes an EDS Section. Check for:
- Background image/video → Section metadata `background` property
- Background variant class → Section metadata `style` property
- Content within `abbv-background-container-content` → Section body content

### Step 3: Map Blocks
For each content element within a section, determine:
- Is it default content (text, images, links)?
- Or a block? (accordion, tabs, cards, formulary, etc.)

Use the selector-to-block mapping from `abbvie-block-library` skill.

### Step 4: Handle ISI (Every Page)
Every page MUST include:
1. Header safety line (in header fragment)
2. ISI block (fragment reference to `/fragments/isi/{brand}`)
3. Floating safety bar (auto-generated by scripts.js)

See `abbvie-isi-migration` skill for details.

### Step 5: Apply Brand Tokens
Ensure the page loads the correct brand tokens:
- Set `brand` metadata in page properties
- Brand tokens at `styles/{brand}/tokens.css`
- Brand theme at `styles/{brand}/themes.css`

### Step 6: Handle Modals
- Exit intent modals → Built into `scripts.js` (auto-detect external links)
- Study design modals → `Modal (study-design)` blocks
- Contact modals → Fragment references

## Linzess Special Case: Hash-Based Navigation

Linzess consolidates 34 URLs into ~12 actual pages using hash anchors:
- `/why-linzess/how-linzess-can-help` → `/why-linzess#howlinzesscanhelp`
- `/understanding-constipation/types-of-constipation` → `/understanding-constipation#typesofconstipation`

For migration: import only the 12 parent pages, each containing all hash-section content.

## Missing Components (Not on Analyzed Pages)

These AEM components exist in the library but were NOT found on the 56 analyzed pages:
- Carousel (`.abbv-carousel`)
- Chart-Pie / Chart-Bar (interactive charts)
- Breadcrumb (`.abbv-breadcrumb-navigation`)
- Doctor Locator (`.abbv-dr-locator`)
- Find-a-Provider (`.abbv-find-a-provider`)
- Banner Ad

Do NOT build parsers for these unless found on additional pages.
