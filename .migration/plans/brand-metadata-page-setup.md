# Multi-brand Token Usage for EDS Blocks — Implementation Plan

## Summary

Yes, this is fully possible and aligns with what we've been building. The approach is:

1. Tokens declared in `styles/{brand}/tokens.css` (single source of truth)
2. Block CSS imports tokens via `@import` (gulp inlines them into compiled output)
3. Block CSS only consumes tokens via `var(--token-name)` — no declarations in blocks
4. Works in UE, localhost, and aem.live because compiled CSS has tokens inlined

## Architecture

```
styles/
  tokens.css                          ← parent defaults (--cdp-* defaults + --be-* defaults)
  skyrizi-hcp/tokens.css              ← brand overrides (--cdp-*, --be-*)
  mavyret/tokens.css                  ← brand overrides (--cdp-*)
  rinvoq-hcp/tokens.css               ← brand overrides (--be-*)

blocks/
  clinical-data-panel/
    clinical-data-panel.css           ← base block CSS (uses var(--cdp-*) only, NO declarations)
    skyrizi-hcp/_clinical-data-panel.css  ← @import base + @import brand tokens + brand styles
    mavyret/_clinical-data-panel.css      ← @import base + @import brand tokens + brand styles

  brand-explorer/
    brand-explorer.css                ← base block CSS (uses var(--be-*) only, NO declarations)
    skyrizi-hcp/_brand-explorer.css   ← @import base + @import brand tokens + brand styles
    rinvoq/_brand-explorer.css        ← @import base + @import brand tokens + brand styles
```

## How It Works

1. **Author sets `brand: mavyret` in metadata** → `aem.js` loads `styles/mavyret/tokens.css` globally
2. **Block CSS loaded** → `blocks/clinical-data-panel/mavyret/clinical-data-panel.css` (compiled by gulp)
3. **Compiled file contains**: base CSS + inlined brand tokens (via `@import`) + brand overrides
4. **All `var(--cdp-*)` resolve** because tokens are inlined in the same file
5. **UE works** because it loads the compiled block CSS which has everything self-contained
6. **No brand set** → `styles/tokens.css` (parent) provides `--cdp-*` defaults globally

## Rules

| Rule | Implementation |
|------|---------------|
| No `--*` declarations in block CSS | ✅ All in `styles/{brand}/tokens.css` or `styles/tokens.css` |
| Block CSS uses `var()` only | ✅ No hardcoded brand values |
| `@import` of tokens in `_block.css` | ✅ Gulp inlines into compiled output |
| UE compatible | ✅ Compiled file is self-contained |
| No duplication | ✅ Single source of truth per brand |
| Brand accuracy | ✅ Values extracted from original sites |
| **No non-CSS files changed** | ✅ Only `.css` files modified |

## Tradeoff

The compiled `blocks/{block}/{brand}/block.css` file will contain a `:root{}` block (inlined from tokens). This is intentional — it's the mechanism that makes UE work without "not defined" errors. The source `_block.css` stays clean.
