XWalk (Universal Editor) Authoring Model
ℹ️
All blocks are authored in the AEM Universal Editor (Cross-Walk / XWalk pattern). Authors see a live WYSIWYG preview. Content is stored as structured JSON in the page model. There is NO document-table authoring (Word/Docs) in this implementation.

component-models.jsonDefines all editable fields shown in UE Properties panel

❌ Global AEM Anti-Patterns Removed in EDS
  • customCssClasses freetext field on every component → authors select from named variants only
  • data-cmp-is attributes driving JS behavior → EDS block class on wrapper drives JS
  • Separate child component types (e.g. accordion-blade, flexboxitem-v2) as droppable parsys children → modelled as model repeater fields inside the parent block
  • Experience Fragment referencePath for modal content → content inline in block model
  • Server-side Sling models / HTL → vanilla ES module JS + CSS only
  • Datasource-driven selects (e.g. ce-icons, ce-mva-value) that require AEM backend → replaced with authored text fields or simplified select with EDS icon names
  • MVA (Most Valued Action) analytics tabs in every dialog → single optional analyticsId text field per block; all GA4/tracking handled by platform-level block
  • Policy-controlled allowed components → component-filters.json controls composition; simpler and transparent
