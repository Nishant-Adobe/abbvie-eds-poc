EDS Block Mapping — AEM Atomic Structure

How each AEM source component maps to EDS blocks using Section + default content + block table patterns.

Hero Block

Source: .abbv-background-container + .hero-home / .hero
Variants: .abbv-background-container-image-swap-bg, .abbv-background-container-video-bg
6 Variations Found:
  • Hero (image-swap) — Background image with responsive swap + overlay text + CTA buttons
  • Hero (video-bg) — Video background with text overlay (RinvoqHCP, Mavyret home)
  • Hero (full-width) — Full-width question hero (Venclexta)
  • Hero (tall) — Tall hero for tab-containing pages (Venclexta financial)
  • Hero (home) — Landing page hero with nav CTAs
  • Hero (interior) — Interior page hero with disclaimer
EDS Authoring:
Section (style: hero) → Background image (via Section metadata) + H1 title + paragraph + CTA button(s)
Video variant: Section (style: hero, video) → Embed block (video URL) + H1 + text
Interior variant: Section (style: hero, interior) → Background image + H1 + disclaimer text

Accordion Block

Source: .abbv-accordion + .abbv-accordion-blade
Variants: .abbv-accordion-single, .abbv-accordion-multi, .accordion-fiesta
5 Variations Found:
  • Single-expand — Only one blade open at a time
  • Multi-expand — Multiple blades can be open (efficacy data)
  • FAQ — Multi-blade with grouped questions (Linzess, Rinvoq DTC)
  • Fiesta — Brand-specific gradient styling (Mavyret, RinvoqHCP)
  • White BG — White background variant (Venclexta)
EDS Authoring:
Accordion block table: Row per blade → Col 1: Blade title | Col 2: Blade content (rich text, images, etc.)
FAQ variant: Accordion (faq) block table with question/answer rows
Single-expand: default behavior. Multi-expand: Accordion (multi)
